一、核心定位

AI Box = 最低门槛的端侧 AI Agent 开发平台。
不是卖一个盒子，而是给开发者一个底座——用最低成本、最快速度，把 AI Agent 从想法变成可部署的产品。

分层叙事

层次 叙事 对应产品
当下落点 端侧 Agent 基座：数据不出盒子、低延迟、离线可靠、即插即用 当前版本（天玑 9300 / 48 TOPS）
未来愿景 具身智能的端侧大脑 下一代（~400 TOPS，路线图可公开）
⚠ "具身智能"仅在 400 TOPS 路线图发布时使用，当前版本不打此标签。

48 TOPS 宣传口径

不比参数大小，强调"绰绰有余"——对于任何行业、任何需求的 Agent 产品开发和验证来说，48 TOPS 端侧算力完全够用。

一、活动概述

1.1 关于 BoxCraft 2026

BoxCraft 2026 是一场面向开发者的端侧 AI Agent 开发挑战赛。

我们相信，AI Agent 不应只存在于云端——它应该离用户更近、响应更快、数据更安全。BoxCraft 2026 邀请你基于 AI Box 端侧硬件平台，打造真正可落地的 AI Agent 应用。

「把 Agent 装进盒子，让智能无处不在」

1.2 关于 AI Box

AI Box 是基于联发科天玑 9300 芯片的端侧 AI Agent 硬件工作站：

参数 规格
核心芯片 联发科天玑 9300
端侧算力 48 TOPS NPU
操作系统 Android
外观 ~10-12cm 方正黑色盒子
接口 USB、触控屏、多种 I/O
软件架构 Android 底层 → 组件抽象层 → Agent 应用层
「不用等更强的芯片——48 TOPS，今天就能做出你的 Agent 产品」

三、参赛规则

3.2 报名流程

  1. 填写报名表：通过赛事官网提交团队信息
  2. 提交项目提案（1-2 页）：选题方向、目标用户、初步方案、如何利用 AI Box
  3. 提案评审（第一轮）：评审委员会书面审核，选出 35-40 支队伍
  4. 预热任务（第二轮）：完成 2-3 个轻量开发任务，验证技术能力
  5. 入选通知：选出 25 支队伍，签署协议并发放 AI Box 设备

3.3 报名审核标准

维度 权重 说明
方向可行性 40% 选题能否在 AI Box 上实现？
团队匹配度 30% 技能覆盖度 + 相关经验
方案创新性 30% 有独立思考和差异化

3.4 作品要求

必须满足（硬门槛）：

  • ✅ 能在 AI Box 或官方模拟环境上运行
  • ✅ 提供清晰的复现步骤（README + 环境说明）
  • ✅ 不触碰敏感数据，不提交密钥/账号
  • ✅ 提交物齐全：代码仓库 + 演示视频 + 路演 PPT + README

鼓励但不强制：

  • 开源代码
  • 编写技术博客分享开发过程
  • 录制开发 Vlog

3.5 AI 辅助编程规则

  • ✅ 允许并鼓励使用 AI 辅助编程工具（Copilot、Cursor、ChatGPT 等）
  • ✅ 需在 README 中披露使用的 AI 工具
  • ✅ 需确保代码可复现、依赖清晰
  • ✅ 需遵守所有引用代码的开源协议

十、FAQ

报名相关

Q：我不是交大学生，可以参赛吗？

A：可以。本赛事面向全国高校学生和自由个人开发者，不限学校、不限地域。

Q：可以一个人参赛吗？

A：可以。团队规模 1-4 人均可。

Q：可以同时报多个赛道吗？

A：一个团队只能报名一个赛道。

技术相关

Q：没有 AI Box 设备怎么开始开发？

A：报名第一轮不需要设备。通过预热任务筛选后，我们会寄送 AI Box。SDK 也提供模拟环境/Docker 方案供前期开发。

Q：可以使用云端 API 吗？

A：可以辅助使用，但作品的核心功能必须能在 AI Box 端侧运行。端侧优势（隐私/低延迟/离线）是评审加分项。

Q：AI 辅助编程可以用吗？

A：鼓励使用。需在 README 中披露工具名称，代码需可复现、可解释，并遵守开源协议。

比赛相关

Q：决赛需要到现场吗？

A：是的。决赛在虹桥国展中心（上海交大机器人年会主会场）进行现场路演。

Q：奖金怎么发？

A：奖金为税前金额，由主办方代扣 20% 个人所得税后银行转账发放。颁奖后 30 天内到账。

已选定 3 个命题方向，均突出 AI Box 端侧优势（隐私/低延迟/离线）。

验收标准 完成至少 5 个故障场景的端到端排查流程（故障描述→诊断→步骤引导→结果记录）

二、评审维度与权重

维度排序：创新性 > 商业价值 > 完成度 > 技术深度 > AI Box 适配度
维度 权重 评审关注点
创新性 25% Agent 方案原创性、差异化亮点、技术路线新颖度、方法突破
商业价值与影响 25% 真实痛点与目标用户清晰；价值可量化；落地/推广/商业化路径清楚
完成度与可用性 20% Demo 端到端可用与稳定；交互清晰；答辩能讲清"为何有效/如何实现"
技术深度 15% 工程结构合理、依赖清晰、可复现、离线兜底、合规披露齐全
AI Box 适配度 15% 体现端侧优势（隐私/低延迟/离线/端云协同/外设）与赛道契合
对学生团队不要求写完整商业计划书。回答四个问题即可：给谁用、解决什么、带来什么提升、怎么部署/推广。

三、Rubric 评分锚点

维度 1 分（不足） 3 分（达标） 5 分（优秀）
创新性 套壳/轻微改动 有明确亮点 有明显突破且可解释
商业价值 痛点泛泛，用户不清 场景具体、价值可量化 有验证迹象或可复制路径
完成度 只能"演"，不稳定 端到端可跑通 体验顺滑且答辩可解释
技术深度 不可复现/依赖混乱 按 README 可复现 低成本复现+离线兜底
AI Box 适配 未体现端侧优势 Box 环境可跑通 端侧优势显著并有优化证据

四、硬门槛（不满足即淘汰）

  • 不触碰敏感数据；不得提交密钥/账号
  • 提交物齐全（README/视频/PPT/仓库），且版本已锁定
